# EventRadar Telegram Bot

Я сохранил ту же основную механику:
- выбор города кнопками;
- выбор категории кнопками;
- показ актуальных событий из Timepad по нажатию;
- админ-панель;
- рассылка;
- история просмотров.

Что изменено под нормальный хостинг:
- убраны захардкоженные токены и id из кода;
- вместо локального MySQL используется SQLite, чтобы проект запускался без отдельной БД;
- таблицы создаются автоматически;
- исправлена работа с фильтрацией событий и категорий через Timepad API;
- добавлен Dockerfile для Railway;
- добавлен .env.example;
- код рассчитан на Railway и PythonAnywhere.

## 1. Что заполнить

Скопируй `.env.example` в `.env` и впиши свои значения.

Пример:

```env
BOT_TOKEN=токен_бота_от_BotFather
TIMEPAD_TOKEN=токен_Timepad
ADMIN_IDS=111111111,222222222
CITIES=Москва,Санкт-Петербург,Новосибирск,Екатеринбург
EVENTS_LIMIT=10
HTTP_TIMEOUT=10
DB_PATH=data/eventradar.db
```

Важно:
- в `ADMIN_IDS` оставь только нужные id админов через запятую;
- если хочешь оставить ровно ту же логику проекта — кнопки и сценарий уже совпадают с исходной идеей.

## 2. Быстрый локальный запуск

```bash
python -m venv .venv
source .venv/bin/activate
pip install -r requirements.txt
cp .env.example .env
python bot.py
```

На Windows вместо `source .venv/bin/activate`:

```bash
.venv\Scripts\activate
```

## 3. Railway — пошагово

### Вариант через GitHub
1. Создай репозиторий и загрузи туда содержимое этой папки.
2. В Railway создай новый проект из GitHub.
3. Railway сам увидит `Dockerfile` и соберёт контейнер.
4. Обязательно создай **Volume** и примонтируй его в `/app/data`.
5. В Variables добавь:
   - `BOT_TOKEN`
   - `TIMEPAD_TOKEN`
   - `ADMIN_IDS`
   - при желании `CITIES`, `EVENTS_LIMIT`, `HTTP_TIMEOUT`, `DB_PATH`
6. Для `DB_PATH` можно оставить `data/eventradar.db`.
7. После деплоя открой Logs и проверь, что бот стартовал без ошибок.

### Что важно на Railway
Если не подключить Volume, SQLite-файл может потеряться при новом деплое или пересоздании контейнера.

## 4. PythonAnywhere — пошагово

### Что лучше выбрать
Для Telegram-бота нужен именно постоянно работающий процесс.
На PythonAnywhere это удобнее делать через **Always-on task**.

### Шаги
1. Загрузи проект в домашнюю папку, например в `~/eventradar-bot`.
2. Открой Bash console.
3. Выполни:

```bash
cd ~/eventradar-bot
python3 -m venv ~/.virtualenvs/eventradar-bot
source ~/.virtualenvs/eventradar-bot/bin/activate
pip install -r requirements.txt
cp .env.example .env
nano .env
```

4. Впиши реальные значения в `.env`.
5. Перейди в **Tasks**.
6. Создай **Always-on task** с такой командой:

```bash
bash -lc 'cd /home/YOUR_USERNAME/eventradar-bot && source /home/YOUR_USERNAME/.virtualenvs/eventradar-bot/bin/activate && python bot.py'
```

7. Сохрани и проверь логи задачи.

## 5. Почему старый код у вас мог не запускаться

Самые частые причины:
- в коде были захардкожены токены и локальная MySQL-конфигурация `127.0.0.1`, которой на хостинге нет;
- в исходнике использовались параметры запроса к Timepad, которые плохо подходят для текущей схемы фильтрации;
- на хостинге не было постоянного процесса или нормальной БД;
- секреты и id были вписаны прямо в файл, из-за чего переносить проект неудобно и небезопасно.

## 6. Если бот запустился, но не показывает события

Проверь по очереди:
1. Верный ли `BOT_TOKEN`.
2. Верный ли `TIMEPAD_TOKEN`.
3. Есть ли логи ошибок в Railway / PythonAnywhere.
4. Выбран ли у пользователя город.
5. Не пустой ли ответ от Timepad по конкретной категории.
